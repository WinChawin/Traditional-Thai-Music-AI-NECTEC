# FindingInstrument

Thai classical music instrument classification from symbolic notation, with emphasis on **interpretability**.

Identifies which of 4 instruments — ขลุ่ย-ปี่ (flute), ฆ้องวงใหญ่ (gong circle), ซออู้ (fiddle), ระนาดเอก (xylophone) — plays a given passage of Thai classical notation, and explains *why*.

## Dataset

| | |
|---|---|
| Source | hand-transcribed scores from a Thai music teacher |
| Pieces | 6 (โหมโรงมหาฤกษ์, สาธุการ, ตับต้นเพลงฉิ่ง, แขกมอญบางช้าง, กล่อมนารี, จีนขิมเล็ก) |
| Instruments | 4 (ขลุ่ย-ปี่, ฆ้องวงใหญ่, ซออู้, ระนาดเอก) |
| Bars | 6,760 (across 24 files) |
| Vocabulary | 22 tokens (7 note bases × 3 octaves + sustain `-`) |

> Original Excel files are **not redistributed** (transcription belongs to the teacher).
> Derived `notes.parquet` (41 KB) is included so the notebooks run end-to-end.

## Method — 4 pillars, 5 notebooks

1. **`01_eda.ipynb`** — Hand-crafted features (sustain ratio, octave distribution, ลูกสบัด density, paren ratio, transition entropy) + token frequency heatmaps + pitch contours + piano-roll plots + bigram transition matrices.

2. **`02_shallow_baselines.ipynb`** — LR / RF / NB / SVM / XGBoost over hand-crafted + n-gram features.

3. **`03_neural_models.ipynb`** — 1D CNN, BiLSTM + attention, small Transformer encoder.

4. **`04_embedding_analysis.ipynb`** — Embedding via UMAP + retrieval (nDCG@k, mAP) + cluster quality (ARI, NMI) + 3 self-supervised variants vs supervised contrastive upper bound.

5. **`05_interpretability.ipynb`** — RF feature importance + permutation importance + SHAP + attention rollout + top discriminative n-grams + feature/token ablation + per-instrument summary cards.

**Cross-validation:** Leave-one-piece-out 6-fold × 3 seeds (`src.utils.make_cv_splits`). Random split is *not* used — the 4 instruments share the same melody source per piece, so random bars would leak across folds.

**Window strategy:** 16-bar windows, stride 8 (50% overlap).

## Three notation quirks worth knowing

1. **Equal heptatonic tuning** (~171 cents/step) — pitch is categorical, not Western chromatic distance.

2. **Paren cells `(...)`** = "this instrument doesn't play this bar" (เครื่องนำเครื่องตาม — leader-follower marker). Distinct from `----` (plays silently). The `has_paren` boolean flag in the dataframe distinguishes them. Across the corpus, 46 bars (~0.7%) are paren-marked and 106 bars (~1.6%) are all-dash.

3. **ลูกสบัด** = bars with ≥5 tokens (multiple notes packed into one sub-beat for ornamental flourish). ~2.5% of bars but instrument-distinctive: ฆ้องวงใหญ่ 98 bars > ขลุ่ย-ปี่ 51 > ระนาดเอก 18 > ซออู้ 5.

**Bar count NOT aligned across instruments** for กล่อมนารี / จีนขิมเล็ก / ตับต้นเพลงฉิ่ง — different scribes, by design. Do not assume bar-by-bar alignment.


If you have the original Excel files, `scripts/normalize_to_processed.py` regenerates `notes.parquet` from `data/interim/` (which is `data/raw/` plus 19 documented typo fixes confirmed by the teacher).

## Notation reference

**Note bases (7):** ด ร ม ฟ ซ ล ท (= Do, Re, Mi, Fa, Sol, La, Ti)

**Octave markers:**
- `ฺ` (U+0E3A, below) = low octave (e.g. `ทฺ` `ลฺ` `ซฺ`)
- (none) = mid octave
- `ํ` (U+0E4D, above) = high octave (e.g. `ดํ` `รํ` `มํ`)

**Per bar:** 1 cell = 4 sub-beats. `-` = sustain or empty sub-beat (not a Western rest).
